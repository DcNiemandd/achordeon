# Editing

When in song edit view, you see the song source on the left side and the render on the right side. You can resize the ratio of the split view.

For mobile view, use button on the sidebar to switch between them.

## Renderer

The song renders from the information you've given it. The information is divided into two parts:

- content - source code, all text that will be rendered
- settings - extra information that changes the render behavior

## Content

You put all text for the song into the editor. The information is labeled by custom markup syntax. You can read about every option on [syntax page](./syntax.mdx).

## Settings

The renderer allows you to customize the output with . You can change it [globally](settings.mdx#rendering) or for this song only.

There are other action buttons to help you with editing:

- insert syntax - buttons for chord, title, and subtitle syntax
- label - gives the current block a label
- transpose up / down - transpose all valid chords
- undo / redo - history only from current session

:::note[Valid chord]
A valid chord is a root + accidental + quality, with an optional `/bass`; brackets that are not a valid chord are never transposed.
:::

:::note
Some actions have keyboard shortcut, i.e. `alt + c` for inserting chord

:::danger
Implement shortcuts for all actions with settings.
:::
