# Editing

When in song edit view, you see the song source on the left side and the render on the right side. You can resize the ratio of these windows.

For mobile view, you see only one window and you can switch between them using button on the sidebar.

:::warning[Source of truth]
The format of input data is **UNSTABLE**.

First version is from PRD app thus is format looks like this. I still haven't decide if it's more important to have all data about the song in the source code or to have more readable source and hide extra info in metadata.
:::

## Renderer

The song renders from the information you've gave. The information is divided in two parts:

- content - source code, all text that will be rendered
- settings - extra information that changes the render behavior

## Content

You put all text for the song into the editor. The information is labeled by custom markup syntax. You can read about every option on [syntax page](./syntax.mdx).

## Settings

The renderer allows you to customize the output:

- scale - custom size for the text, useful for maximizing the page space
- columns - for really long songs
- title position - you can have title on top of the page or on the side
- aspect ratio - overwrites global setting

There are other action buttons, that helps you with the editing:

- insert syntax - buttons for chord, title, and subtitle syntax
- label - gives the current block a label
- transpose up / down - transpose all valid chords
- undo / redo - history only from current session

:::note
Some actions have keyboard shortcut, i.e. `alt + c` for inserting chord
:::

## Song actions

You can call same actions as from the list:

- favorite
- download
